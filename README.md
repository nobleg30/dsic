# DSIC Scoreboard – UNIC-CASS IC Submission

This repository contains the implementation of the **dsic_scoreboard** digital integrated circuit prepared for fabrication using the **UNIC-CASS wrapper framework**.  
The design flow follows the open-source ASIC toolchain using **LibreLane** and the **IHP Open PDK** to generate a hardened macro and integrate it into the UNIC-CASS chip wrapper.

---

# 4. Repository Organization

The repository contains the UNIC-CASS wrapper framework along with the user macro implementing the **dsic_scoreboard** design. The main directories are arranged as shown below:

```
├── unic_cass_wrapper_user_project/
│   └── dsic_scoreboard/
│       ├── src/
│       │   ├── dsic_scoreboard.v
│       │   └── user_project_example.v
│       ├── config.json
│       └── final/              # Generated by the flow (not tracked in git)
│
├── unic_cass_wrapper/
│   ├── src/
│   │   └── user_project_wrapper.sv
│   ├── config.json
│   └── final/                  # Generated during chip-level implementation
│
├── IHP-Open-PDK/
├── librelane/
├── Makefile
└── README.md
```

The `final/` directories are created automatically when the design flow is executed. These folders store the generated implementation files such as GDS, LEF, reports, and extracted data, and are therefore excluded from version control through `.gitignore`.

# 5. Environment Preparation

The complete ASIC flow for this project is executed inside the **UNIC-CASS Docker environment**, which includes the necessary tools and dependencies.

After cloning the repository, move into the project directory:

```bash
git clone https://github.com/nobleg30/dsic.git
cd unic-cass-wrapper
```

Next, initialize the project dependencies:

```bash
make setup
```

Running this command downloads and configures the required components, including:

- **LibreLane** (automated ASIC flow)
- **IHP Open PDK**
- supporting infrastructure for the **UNIC-CASS wrapper**

---

# 6. Hardening the `dsic_scoreboard` Macro

The **dsic_scoreboard** module is synthesized and physically implemented as a macro using the LibreLane flow.

Navigate to the user project directory:

```bash
cd unic_cass_wrapper_user_project
```

Start the hardening process with:

```bash
make dsic_scoreboard
```

During this stage, the RTL is transformed into a physical layout through multiple automated steps such as synthesis, floorplanning, placement, clock tree synthesis, routing, and signoff analysis.

Upon completion, the flow generates several output files including:

- GDS layout files
- LEF abstract views
- timing libraries
- extracted parasitic data
- implementation reports

To open the generated results for inspection:

```bash
make dsic_scoreboard VIEW_RESULTS=1
```

---

# 7. Integration with the UNIC-CASS Wrapper

Once the macro implementation is completed, the design is incorporated into the **UNIC-CASS chip wrapper** for full chip assembly.

Move to the wrapper directory:

```bash
cd ../unic_cass_wrapper
```

Run the top-level implementation flow:

```bash
make
```

This stage performs the chip-level physical design tasks, including:

- chip floorplanning  
- pad ring construction  
- power grid generation  
- placement of the user macro  
- global and detailed routing  
- manufacturability verification

The process concludes at the **Report Manufacturability** stage, producing the layout required for the mock tapeout.

---

# 8. Viewing the Generated Layout

The resulting layout files were analyzed using **KLayout**, an open-source layout viewer widely used in VLSI design.

Example commands to view the generated GDS files:

```bash
klayout final/gds/user_project_wrapper.gds
klayout unic_cass_wrapper_user_project/dsic_scoreboard/final/gds/user_project_example.gds
```

When examining the layout:

- Lower metal layers and internal routing are useful for inspecting the **macro implementation**.
- Upper metal layers and pad structures are more relevant when observing the **complete wrapper layout** prepared for the mock tapeout.
